this指向问题

###首先明白：

1：this永远指向一个对象；

2：this的指向完全取决于函数调用的位置；

但是，由于js语言之中，一切皆为对象，所以看函数在那个运行对象下进行，但是js支持运行环境动态切换，所以this的指向是动态的

1、分析

首先，在JS中，数组、函数、对象都是引用类型，在参数传递时也就是引用传递；

样例：

function foo() {
    console.log(this.a);
}
var obj2 = {
    a: 2,
    fn: foo
};
var obj1 = {
    a: 1,
    o1: obj2
};
obj1.o1.fn(); // 2

obj1对象的o1属性值是obj2对象的地址，而obj2对象的fn属性的值是函数foo的地址；

，我们对this使用最频繁的几种情况做一个总结，最常见的基本就是以下5种：

对象中的方法，事件绑定 ，构造函数 ，定时器，函数对象的call()、apply() 方法；

2、事件绑定

事件绑定共有三种方式：行内绑定、动态绑定、事件监听；

###（1）行内绑定

解释：

行内绑定事件的语法是在html节点内，以节点属性的方式绑定，属性名是事件名称前面加’on’，

属性的值则是一段可执行的 JS 代码段；而属性值最常见的就是一个函数调用；

情况：

<input type="button" value="按钮" onclick="clickFun()">
<script>
function clickFun(){
    this // 此函数的运行环境在全局window对象下，因此this指向window;
}
</script>
<2>​
<input type="button" value="按钮" onclick="this">
<!-- 运行环境在节点对象中，因此this指向本节点对象 -->

由于事件触发时，属性值会作为js代码执行，当，

情况一：

由于当前运行环境无次函数，所以浏览器会跳出当前环境，直接在整个环境找clickFun函数，所以此时指向windows对象

情况二：

没有任何函数，所以直接在当前节点调用

###（2）动态绑定与事件监听

因为动态绑定的事件本就是为节点对象的属性(事件名称前面加’on’)重新赋值为一个匿名函数，因此函数在执行时就是在节点对象的环境下，this自然就指向了本节点对象；

例：

<input type="button" value="按钮" id="btn">
<script>
var btn = document.getElementById('btn');
btn.onclick = function(){
    this ;  // this指向本节点对象，即为input
}
</script>

事件监听中this指向的原理与动态绑定基本一致，所以不再阐述；

3、构造函数中的this

例子：

function Pro(){
    this.x = '1';
    this.y = function(){};
}
var p = new Pro();

第三步强制this指向新建的对象

4、windows定时器下的this

例子：

var obj = {
    fun:function(){
        this ;
    }
}
​
setInterval(obj.fun,1000);      // this指向window对象
setInterval('obj.fun()',1000);  // this指向obj对象

在上面的代码中，setInterval(obj.fun,1000) 的第一个参数是obj对象的fun

因为 JS 中函数可以被当做值来做引用传递，实际就是将这个函数的地址当做参数传递给了 setInterval 方法，

换句话说就是 setInterval 的第一参数接受了一个函数，那么此时1000毫秒后，函数的运行就已经是在window对象下了，也就是函数的调用者已经变成了window对象，所以其中的this则指向的全局window对象；

但是

setInterval(‘obj.fun()’,1000) 中的第一个参数，实际则是传入的一段可执行的** JS 代码**；

1000毫秒后当 JS 引擎来执行这段代码时，则是通过 obj 对象来找到 fun 函数并调用执行，那么函数的运行环境依然在 对象 obj 内，所以函数内部的this也就指向了 obj 对象；

这里需要注意的是，函数后面要是没有加（），表示传值，也就是传址，而加上才等于加一段js代码

5、函数对象的call()、apply() 方法

这两个方法接收对象作为参数，来指定this指向，但是却改不了箭头函数中的this

（1）call

使用：

call(obj,arg1,arg2...argN);

参数说明:

obj:函数内this要指向的对象,

arg1,arg2…argN :参数列表，参数与参数之间使用一个逗号隔开

var lisi = {names:'lisi'};
var zs = {names:'zhangsan'};
function f(age){
console.log(this.names);
console.log(age);

}
f(23);//undefined
    ​
//将f函数中的this指向固定到对象zs上；
f.call(zs,32);//zhangsan

（2）apply

函数名称.apply(obj,[arg1,arg2…,argN])

参数说明:

obj :this要指向的对象

[arg1,arg2…argN] : 参数列表，要求格式为数组

var lisi = {name:'lisi'}; 
var zs = {name:'zhangsan'}; 
function f(age,sex){
console.log(this.name+age+sex); 
}
//将f函数中的this指向固定到对象zs上；
f.apply(zs,[23,'nan']);

注意：call和apply的作用一致，区别仅仅在函数实参参数传递的方式上；

这个两个方法的最大作用基本就是用来强制指定函数调用时this的指向；

arguments对象

前提：函数调用时。浏览器传递的隐式参数

函数的上下文对象this、

封装实参的对象arguments

1、arguments对象理解

官方解释：

是一个对应于传递给函数的参数的类数组对象，

理解：即其中包含了所有的函数参数，且也可以进行参数的修改

样例：

function fun(a,b,c){
console.log(arguments[1])  //2
arguments[1]='23'
console.log(this.bb)    //undefined
console.log(b)     //23
}
fun(1,2,3)

注意，第二个undefined由于：当函数不是对象属性时（即是函数而不是方法），this被绑定为全局对象，即为windows对象。

2、性质分析

概述：

arguments 对象实际上是所在函数的一个内置类数组对象

每个函数都有一个arguments属性，表示函数的实参（即外面给里面的实际的数）集合，这里的实参是重点，就是执行函数时实际传入的参数的集合。arguments不是数组而是一个对象，但它和数组很相似，所以通常称为类数组对象，以后看到类数组其实就表示arguments。arguments对象不能显式的创建，它只有在函数开始时才可用。

arguments还有属性callee，length和迭代器Symbol。

arguments同样具有length属性，arguments.length 为函数实参个数，可以用arguments[length]显示调用参数

它可以让函数调用数量不定的参数，更灵活了

arguments对象可以检测参数个数，模拟函数重载

3、特殊

（1）严格模式

function f3(a,b){
  'use strict';
  arguments[0] = 10;
  arguments[1] = 20;
  return a + b;
 }

 console.log(f3(3,6)); //9

（2）callee属性可以返回对应的原函数，达到调用自身函数的效果，但是在严格模式中是不适用的

var f5 = function(){
   console.log(arguments.callee===f5); //true
   console.log(arguments.callee===f6); //false
  }

  var f6;
  f5(); //返回原函数，调用自身

这里的严格模式，直接报错

4、应用

（1）判断形参实参是否一致

function add(a, b) { 

var realLen = arguments.length; 
console.log("realLen:", arguments.length);   //5
var len = add.length; 
console.log("len:", add.length);  //2

if (realLen == len) { 
console.log('实参和形参个数一致'); 
} else { 
console.log('实参和形参个数不一致');   //不一致
} 
   console.log(arguments[4])  //66
   }; 

   add(11,22,33,44,66);

注意，由于argument类数组存在的原因，就是不一样，没有传的实参也可以用它就行调用

###（2）callee实现递归

var sum = function(n) { 

 if(n == 1) { 
  return 1; 
 } else { 
  return n + arguments.callee(n-1);

　 }
} 

console.log("sum =", sum(5));   //15

（3）编写一个函数求传入的若干个数字（不能用数组显示传入）的和（无形参）

function add() { 

var len = arguments.length; 
var sum = 0; 

for (var i = 0; i < len; ++i) { 
  sum += arguments[i]; 
} 
return sum; 
}; 

add(11,22,33,44,66);

4、对象属性

（1）constructor属性（指向构造函数）

返回对创建此对象的数组函数的引用。

function fun(){
 age=12
}
var s = new fun()   //这个感觉数组与对象都报错了
console.log(s.constructor == fun)    //true
console.log(s.constructor == fun.prototype.constructor)  //true
主要由于两者都指向原型对象
console.log(s == fun.prototype)   //false

所以可以理解为：

在Javascript 语言中，constructor 属性是专门为 构造函数 而设计的，它存在于每一个function 的prototype 属性中。这个constructor 保存了指向 原型的prototype。

除非进行修改，否则就是实例中的属性定义也不会改变：

var ss = new fun()
<!-- 实例的属性改变,覆盖了原型中的属性 -->
ss.name = 'make'
console.log(ss.constructor == fun.prototype.constructor)  //true
fun.prototype = {
    age:45,
    name:'sad'
}
console.log(ss.constructor == fun.prototype.constructor)  //false

重写构造函数的原型之后，再创建的实例才会用新的原型，否则之前的实例会引用最初的，但是当原型改变，并未重写时，那么之前的实例也会同步

解释：

当作为fun作为构造函数来创建对象时，对象实例的内部自动保存一个指向其构造函数（这里就是fun）的 prototype 对象的一个属性

所以我们在每一个对象实例中就可以访问构造函数的 prototype 所有拥有的全部属性和方法，就好像它们是实例自己的一样。当然该实例也有一个 constructor属性了（从 prototype 那里获得的），每一个对象实例都可以通过 constrcutor 对象访问它的构造函数，

同理：instanceof似乎判断更好

function fun(){
 age=12
}
var s = new fun()  
console.log(s instanceof fun)

（2）__proto__属性（指向prototype）

首先最值得注意的就是前后是两条下划线_ _

具体的指向该对象的原型，从任何对象沿用它开始遍历都可以追溯到Object.prototype。其实主要由于实例的内部[[prototype]]无法直接访问，只能通过它来访问对象原型

function fun(){
 age=12
}
var s = new fun()  
console.log(s.__proto__==fun)  //false
console.log(s.__proto__==fun.prototype)  //true
注意这里返回的是原型对象prototype，而fun是函数
console.log(s.__proto__.constructor ==fun)   //true

（3）prototype属性（访问到原型prototype）

用法：prototype 属性使您有能力向对象添加属性和方法。

object.prototype.name=value

实例：

function employee(name,job,born)
{
this.name=name;
this.job=job;
this.born=born;
}

var bill=new employee("Bill Gates","Engineer",1985);

employee.prototype.salary=null;  //直接在原型上添加，但是这里直接写也一样
bill.salary=20000;     //构造的对象也可以使用

document.write(bill.salary);

联系

对象都有__proto__属性，而且他指向的就是对象的原型。

function Person() {
}
var person = new Person();
console.log(person.__proto__ === Person.prototype); // true

联系图

我们读取一个属性的时候，如果在实例属性上找到了，就读取它，不会管原型属性上是否还有相同的属性，这其实就是属性屏蔽，即当实例属性和原型属性拥有相同名字的时候，实例属性会屏蔽原型属性，记住只是屏蔽，不会修改，原型属性那个值还在；但是如果在实例属性上没有找到的话，就会在实例的原型上去找，如果原型上还没有，就继续到原型的原型上去找，直到尽头。

这个尽头是啥？由于原型也是对象，所以也会有__proto__属性，也就是原型的原型；最后就可以找到Object.prototype这个大boss，所有原型对象都是Object构造函数生成的（Object.prototype值为null）。正是因为所有的原型最终都会指向Object.prototype，所以对象的很多方法其实都是继承于此，比如toString()、valueOf()，前面用到的hasOwnProperty，甚至是.constructor、proto

样例：

function fun(){
 age=12
}
var s = new fun()  
console.log(fun.prototype.__proto__ == Object.prototype)  //true

4、promise函数，解决回调地狱

情景：

有js文件，要求依次读取a.txt , b.txt , c.txt

当没有任何读取限制时：

var fs = require('fs')

 进行异步读取
 fs.readFile('./a.txt','utf-8',function(err,data){
     console.log(data)
 })
 fs.readFile('./b.txt','utf-8',function(err,data){
     console.log(data)
 })
 fs.readFile('./c.txt','utf-8',function(err,data){
     console.log(data)
 })

 由于是异步操作,所以那个执行完(读的快),打印那个
 所以这样的执行结果不确定

所以采取回调函数的方法

fs.readFile('./a.txt','utf-8',function(err,data){
        console.log(data)
        fs.readFile('./b.txt','utf-8',function(err,data){
                console.log(data)
                fs.readFile('./c.txt','utf-8',function(err,data){
                        console.log(data)
                    })
            })
    })
// 但是老是这样套会降低代码的可读性,多次嵌套造成造成"回调地狱"

promise读取一个文件时：

promise传入了一个匿名函数,并传入两个参数,res,rej代表了两个函数

var p1 = new Promise(function (res, rej) {
    fs.readFile('./a.txt', 'utf8', function (err, data) {
        if (err) {
            // rej代表失败
            rej(err)
        } else {
            // 代表成功就会调用这个函数
            res(data)
        }
这里可以理解为res,rej是为了给后面的then操作提供数据的
    })
})
// then的两个参数,一个代表成功一个代表失败
p1.then(function (data) {

成功,这个成功的参数可以与上面匿名函数的参数不一样,但是上面必须传参数

    console.log(data)
}, function (err) {
    console.log(err)
})
console.log(p1)

实际解决问题:

var fs = require('fs')

function fun(files) {
    return new Promise(function (res, rej) {
        fs.readFile(files, 'utf8', function (err, data) {
只对成功的结果进行传值
            res(data)
            
        })
    })
}
// 通过链式操作来解决回调的问题

fun('./a.txt').then(function (data) {
    console.log(data)
    return (fun('./c.txt'))
    // 由于这里返回'./b.txt',所以这个then执行完后,相当于变为
    // fun('./b.txt').then,又创建了一个容器对象promise
}).then(function (data) {
    console.log(data)
    return (fun('./b.txt'))
}).then(function(data){
    console.log(data)
})

Promise原理

promsie对象类似有一个容器(new之后就会被建立),异步与代码在此执行,且异步的执行必须有两个状态要么成功要么失败.

且这两种状态不可逆,就是如果代码失败,后面一定会执行then后的第二个参数,也就是执行第二个函数,这就是文档管理,管理异步

每一个异步都需要一个独立的promise容器进行管理,也都要创建一个promise容器,

#2、module.exports与exports，export与export default之间的关系和区别

a、commonJS规范：

Node应用由模块组成，采用CommonJS模块规范。根据这个规范，每个文件就是一个模块，有自己的作用域。在一个文件里面定义的变量、函数、类，都是私有的，对其他文件不可见。

CommonJS规范规定，每个模块内部，module变量代表当前模块。这个变量是一个对象，它的exports属性（即module.exports）是对外的接口。加载某个模块，其实是加载该模块的module.exports属性。

导入：require（’aaa’）；

导出：module.exports = abc;

样例：

const aaa = require('aaa');
b={name:aaa}
...
module.exports=b

module.exports与exports区别

exports只能使用语法来向外暴露内部变量：如http://exports.xxx = xxx;

module.exports既可以通过语法，也可以直接赋值一个对象。

但是exports = module.exports，两者的堆内的指向是一样的例如：

console.log(module.exports === exports);

//输出结果为：true

输出结果是true其实就说明它们就是一个东西，其实exports = module.exports，因为他们是引用类型的一个变量名，所以当exports再指向一个引用类型的时候，那么他们就不再全等。

exports = [0, 1];
console.log(exports === module.exports);

//输出结果为：false

当然，如果直接通过http://exports.xxx的形式赋值，那么他们依然会指向同一个地址：

exports.array = [0, 1];
console.log(exports === module.exports);

//输出结果为：true

但是

如果对exports直接赋值，会导致exports的指向不一样了，所以在其他文件中无法找到其传递的东西。

而module.exports是在执行模块代码中就将module传入到了函数中，所以即使module.exports的值改变也能够在其他文件中进行调用。

但是此时exports.xxx就无法使用了

export {m};

c、export default 和 export区别

首先注意export、import可以有多个，export default仅有一个

通过export方式导出，在导入时要加{ }，export default则不需要

1.export

//a.js
export const str = "blablabla~";
export function log(sth) { 
  return sth;
}
对应的导入方式：

//b.js
import { str, log } from 'a'; //也可以分开写两次，导入的时候带花括号

2.export default

//a.js
const str = "blablabla~";
export default str;
对应的导入方式：

//b.js
import str from 'a'; //导入的时候没有花括号

且注意：

使用export导出的成员，必须严格按照导出时候的名称，来使用{ }按需接收，export defaultb=不用加

名字必须一样！！！！

使用export导出的成员，如果想换个变量名称接收，可以使用as来起别名

// test.js
var info = {
name: 'zs',
age: 20
}
export default info

export var title = '小星星'

export var content = '哈哈哈'

导出

import person, {title, content as content1} from './test.js'
console.log(person);
console.log(title + '=======' + content1);

3、模块加载顺序

当地址没有路径（没有.\）的时候，要么是核心模块，要么是第三方模块（如moment）

查找顺序为，先核心模块再第三方模块，第三方模块位置在node_modules（同等级目录下）下，找同名文件夹

当然了如果同名文件夹找不到直接报错！！！

之后再找package.json，这里面的main属性规定了要加载谁谁谁，没有这个属性或者这个文件就找index.js之后是index.json,index.node

当然了如果当前目录没有node_modules，就默认去上一层目录，直到根路径为止，

注意：

package.json是确定要加载哪一个文件，如果没有这个文件，那个就去加载index.js

6、HTTP协议

http的整个协议是在tcp基础上建立起来的，且第一次请求的三次握手（客户端与服务器）可以理解为

一次：你在吗 — 我在

二次： 我要发信息了 — 发吧

三次： 发送信息 – 接收

并且我们的http请求在发送数据时，有很多次发数据（一次发不完），所以服务器在接收时，为了知道http什么时候发送数据，node中建立了事件（客户端发送数据的事件）绑定。
